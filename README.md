# 2D Fighting - 2D Игра

Проект, созданный для Яндекс Лицея. Это 2D игра в жанре платформер с элементами боевой системы, где игрок должен пройти уровни, победить противников и может сразиться с другом в локальном режиме.

## Возможности

- **Кампания из 6 уровней** с возрастающей сложностью
- **Локальный мультиплеер** - режим для двух игроков на одном компьютере
- **Система скинов** - выбор персонажей и противников
- **Таблица лидеров** - отслеживание лучших результатов
- **Система пройденных уровней** - сохранение прогресса
- **Настраиваемый внешний вид** - выбор фонов и персонажей

## Технологии

- **Python 3** - основной язык программирования
- **Pygame** - графическая библиотека для создания игры
- **SQLite3** - база данных для хранения данных пользователей и прогресса

## Установка и запуск

1. Убедитесь, что у вас установлен Python 3.7+
2. Установите необходимые зависимости:
```bash
pip install pygame
```

3. Запустите игру:
```bash
python game.py
```

## Управление

### Для игрока 1 (в одиночной игре и мультиплеере):
- **A** - движение влево
- **D** - движение вправо
- **W** - прыжок
- **Пробел** - обычная атака
- **Ctrl + Пробел** - усиленная атака

### Для игрока 2 (в мультиплеере):
- **Стрелка влево** - движение влево
- **Стрелка вправо** - движение вправо
- **Стрелка вверх** - прыжок
- **Enter** - обычная атака
- **Правый Ctrl + Enter** - усиленная атака

## Структура проекта

- `game.py` - основной файл игры
- Папка `game_data/` содержит:
  - Изображения персонажей, фонов и интерфейса
  - Базу данных SQLite `game_db.db`
  - Музыкальное сопровождение `music.mid`

## Особенности игры

### Система персонажей
- 6 игровых персонажей на выбор (рыцарь, смурф, фея, ковбой, Гарри Поттер, герой)
- 6 типов противников (призрак, ведьма, скелет, эльф, чумной доктор, каменный человек)
- 4 варианта фоновых изображений

### Система уровней
- Уровни становятся сложнее: противники получают больше здоровья и наносят больше урона
- Прогресс сохраняется в базе данных
- Награда за прохождение уровня зависит от оставшегося здоровья

### Мультиплеер
- Локальный режим для двух игроков
- Раздельное управление для каждого игрока
- Возможность выбора любого персонажа

## База данных

Игра использует SQLite базу данных со следующими таблицами:

- `users` - данные пользователей (id, имя, пароль, счет)
- `levels` - информация о пройденных уровнях (id пользователя, номер уровня)
